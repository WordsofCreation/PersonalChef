# Public Pricing Guidance Notes

Phase 5 adds a customer-safe pricing presentation layer for the public Sample Menu Builder flow.

## Purpose

The public estimate guidance is designed to help guests set expectations without exposing internal pricing mechanics.

## Safe public fields

The following fields are safe to render in the public UI:

- `heading`
- `tierLabel`
- `tierDescriptor`
- `estimateLabel`
- `rangeLow`
- `rangeHigh`
- `startingAtPerPerson`
- `summary`
- `supportingText`
- `disclaimer`

These fields are generated by `src/data/publicPricingGuidance.ts` and are intentionally phrased as planning guidance rather than a binding quote.

## Private fields that must remain unpublished

Do not render or serialize any raw internal pricing detail, including:

- food cost rollups
- labor assumptions or formulas
- overhead or travel calculations
- markup or minimum-booking logic
- internal pricing notes or strategy details

Examples of private fields include:

- `estimated_total_food_cost`
- `estimated_cost_per_person`
- `estimated_labor_cost`
- `estimated_shopping_admin_cost`
- `estimated_travel_cost`
- `estimated_overhead_cost`
- `estimated_internal_total_cost`
- `suggested_client_price`
- `pricing_notes`
- `tier_notes`

## Fallback behavior

When the builder state is too incomplete to support useful guidance, the pricing block stays hidden.

When the menu suggests a more customized celebration-style proposal and guest count is not yet known, the public layer falls back to a softer custom-proposal message instead of a rigid range.

## Current public behavior

- Menu preview shows an elegant planning guidance block beneath the composed menu preview.
- Contact flow can carry the safe estimate summary forward with the saved menu request.
- Guest count remains a refinement input for the formal proposal flow rather than a public calculator requirement.
